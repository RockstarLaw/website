# RockStar Law — Design Rules

These are locked design constraints. Do not modify or expand without explicit approval.

## Brand

- **Brand name in text:** RockStar Law (capital S, capital L). Never "Rockstar Law" or "ROCKSTAR LAW" in body copy.
- **Tagline (always two lines, in this order):**
  - Train Here. Litigate Anywhere.
  - Graduate Courtroom Ready.

## Core Site vs. Module Sites

There are two distinct visual treatments:

**Core site** (homepage, About, Why RockStar Law, Pricing, etc.)
- Modern, editorial, light, premium SaaS feel
- Inspiration: Upwork /hire page

**Module simulation pages** (StarBiz, USPTO, eCO, EDGAR, IRS, Courts)
- Faithfully replicate the actual government/agency UX
- The "ugly UX" is intentional — fidelity is the product
- This is the ONLY context where boxed/dated/government-style is correct

The rules below apply to the **core site only**.

## Color Palette

- **Background:** white. Never dark theme on the core site.
- **Accent:** Primary Red — the design language of the site
- **Neutrals:** Cold gray (not warm), black
- No additional colors. No gradients. No theme drift.

## Red as a Design Element (UI)

Red is the visual signature of RockStar Law. Use it as a design tool throughout the site, not sparingly. Confirmed uses include:

- Primary call-to-action buttons (Get Started, etc.) — solid red background, white text, semibold
- Short red underlines beneath section headings
- The three-dot ellipsis separator (`. . .`) rendered in red
- Red outline on button hover/focus states
- Red arrowhead bullets in lists (→)
- Red accent lines, dividers, and key separators
- Active/selected states, hover accents

Expand red usage anywhere it strengthens the look. The constraint is taste, not quantity. Red is what makes the site feel like RockStar Law — lean into it.

The only place red is constrained to a single instance is inside images, per the next section.

## Red in Images (The Hero Marker Rule)

Inside images and photographs, red is signal — not decoration. Red identifies the hero of the scene.

**Object/abstract images** (pencils, microphones, files, etc.)
- Exactly one red element per scene
- Examples: a red microphone cable among gray vintage mics; a single red pencil among yellow pencils

**People images**
- Red marks the hero of the scene
- One red marker per hero
- Multiple heroes allowed in the same scene = multiple red markers (rare, controlled, never random extras)
- **Female hero marker:** designer-style red high heels (Christian Louboutin energy — visible, deliberate)
- **Male hero marker:** red tie OR red pocket square OR red interior of flipped cuff. Pick one. Never stack.

UI red and image red obey different rules. UI red is the brand design language and is used freely. Image red is constrained to the hero marker rule above.

## Image Production Rules

- **Source:** All images must be generated specifically for RockStar Law. Never use photos from the internet, stock libraries, or any external source. Every image on the site is created new.
- **Aspect ratio:** square (1:1) for section images
- **Background:** must blend into the page's white background — no rectangular edges visible
- **No frames, borders, cards, or shadows** around images
- **Style:** photoreal but elevated — "the real world if that world were perfect around the heroes." Non-hero elements can look antiquated or ordinary.
- **Filenames:** lowercase with dashes, scoped by purpose (e.g., `about-operational-fluency-female.png`, `why-rockstar-law-operator.png`)

## Layout

- **Editorial flow, not boxed sections.** No cards, panels, borders, or heavy shadows on the core site.
- **Section headings:** centered, with a short primary-red underline beneath the title (not full width)
- **Ellipsis separators (`. . .`):** rendered in red, with tight spacing above and below; never overlapping with headings or text
- **Vertical rhythm:** consistent spacing between sections; deliberate breathing room without dead air

## Typography

- **Do not increase text size.** This rule is global and persistent.
- Hierarchy is achieved through weight, position, and structure — not size escalation.
- Body text remains readable but tight; never oversized line-height or paragraph spacing.

## List & Module Styling

- **Lists:** red arrowhead bullets (→), not numbered bullets, not standard dots
- **Module row format:**
  - **Bold module label** (e.g., `U.S. Copyright Office.`)
  - Indented description on the next line, normal weight

## Footer

- Light/cold gray background
- Black or dark gray text, no boxes
- Includes legal links and copyright line

## What to Never Do

- Never use images sourced from the internet, stock libraries, or any external source — all images must be newly generated
- Never use boxed cards, framed images, or heavy shadows on the core site
- Never increase font size to add emphasis
- Never introduce new colors beyond the palette
- In images, never use red as decoration — red in images is reserved for the hero marker
- In a single image, never give a non-hero a red item
- Never use generic stock-photo framing
